# 搬瓦工更换 IP：付费换 IP 全流程 + 如何选对套餐

## 一、换 IP 前先确认问题在哪里

建议至少做三组检查：

1. 登录 Client Area / KiwiVM，确认 VPS 状态是 Running，并查看负载和流量是否异常。
2. 分别从当前网络、手机热点和境外探测点测试 IP。只有某一个网络不通，更像路由或本地网络问题；多个地区均不通，则要继续检查 VPS 状态和防火墙。
3. 在 KiwiVM 使用 Serial Console 或重启后仍无法连接时，检查 SSH 服务、监听端口、系统防火墙和安全组规则。

详细排查见：[搬瓦工 IP 被封了吗？判断清单](./ip-blocked-check.html)。

## 二、搬瓦工付费换 IP 的完整流程

### 第 1 步：打开官方申请页

登录搬瓦工账号后访问 [Request IP Change](https://bandwagonhost.com/ipchange.php)。如果页面要求登录，这是正常现象。

### 第 2 步：核对 VPS 实例

根据实例、当前 IP 和机房确认目标 VPS。不要仅凭列表顺序选择，避免给错误的实例换 IP。

### 第 3 步：阅读实时价格和条件

官网会显示当前费用与适用条件。参考文章中的 `$8.79` 不能当作长期固定价格；**以你登录后看到的报价为准**。付款前同时确认：

- 新 IP 是否由系统分配，通常不能自选；
- 旧 IP 是否会立即回收；
- 当前套餐是否已经包含免费换 IP 权益；
- 更换后 DNS、SSH known_hosts、反向解析和业务白名单是否需要更新。

### 第 4 步：支付并等待处理

完成订单后，在 Client Area、工单或邮件里查看状态。不要在处理过程中重复下单。

### 第 5 步：用新 IP 恢复服务

拿到新 IP 后依次处理：

1. 更新域名 A 记录和 CDN 源站；
2. 更新防火墙、数据库、第三方 API 的 IP 白名单；
3. 删除本机 `known_hosts` 中旧主机指纹记录，再核对新指纹；
4. 检查 rDNS/PTR；
5. 从多个网络验证 SSH、网站和业务端口。

## 四、如何选套餐，才能降低换 IP 的长期成本

先按需求排序，而不是先看最贵的方案：

| 你的真实需求 | 优先检查 | 不应默认 |
|---|---|---|
| 预算最低、仅跑轻量服务 | 年付总价、内存、流量、当前库存 | 便宜方案一定有免费换 IP |
| 中国方向网络体验 | 目标地区实测、线路说明、晚高峰表现 | CN2 GIA 能保证 IP 永不受限 |
| 经常需要更换 IP | 产品页是否明确写免费换 IP及间隔 | 所有高端套餐都有此权益 |
| 需要迁移机房 | 可迁移地点、迁移次数、数据风险 | 迁移等于无条件免费换 IP |
| 核心生产业务 | 备份、监控、恢复方案、SLA | 单台 VPS 或单个 IP 足够高可用 |

线路影响网络路径与延迟，但没有官方证据表明“更贵线路的 IP 天然不会被封”。真正能直接计算的，是套餐价格、可迁移地点和明确写入产品页的换 IP 权益。

## 五、换完 IP 后为什么仍可能连不上

- 域名 DNS 缓存仍指向旧 IP；
- 本地 SSH 保存了旧主机指纹；
- 服务只监听旧地址或 `127.0.0.1`；
- 防火墙、授权系统或第三方白名单未更新；
- 你的本地网络或运营商路由仍异常；
- VPS 本身的系统或服务故障没有因换 IP 而消失。

因此，换 IP 是地址层面的处理，不是所有网络故障的万能修复。
