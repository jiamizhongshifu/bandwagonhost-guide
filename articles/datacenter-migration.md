迁移机房和单独申请换 IP 是两个不同操作。跨机房迁移通常伴随网络与地址变化，但是否可迁移、目标机房、冷却时间和最终 IP 都取决于你的具体套餐与 KiwiVM 当时显示的选项。

## 迁移前先做四件事

1. 在 KiwiVM 创建可用备份或快照，并确认关键数据另有副本；
2. 降低 DNS TTL，记录旧 IP、rDNS 和所有白名单；
3. 检查套餐允许迁移到哪些机房以及是否收费；
4. 预留业务中断窗口，不在重要活动前临时迁移。

## 迁移后检查

- 更新域名 A 记录、CDN 源站和监控目标；
- 更新第三方服务、数据库和防火墙白名单；
- 检查系统防火墙是否仍引用旧 IP；
- 重新核对 SSH 主机指纹和 rDNS；
- 从主要用户所在网络实测延迟、丢包和业务端口。

如果你的目的只是处理疑似 IP 连通问题，不要先假设迁移一定可用或一定免费。先做[IP 被封判断](./ip-blocked-check.html)，再根据套餐权益选择迁移或[申请更换 IP](./ip-change.html)。
